## Verdict

**B.6 negative test: PASS**

The coverage gate correctly distinguishes between schema-compliant and schema-violating code, and exits non-zero when a violation is present. This is the contract required by ADR-0015 §Decision: "explicitly waives runner-side kind enforcement for now ... fails CI on host-side schema drift".

## Coverage gate design also exercises violations internally

In addition to this manual negative test, `scripts/observability-coverage.test.ts` itself contains 6 fixture-based negative tests that codify the same contract:

- `fails forward validation for an unknown namespace`
- `fails kind validation when a host span omits openinference.span.kind`
- `fails backward validation when a keep/rename target is missing`
- `fails backward validation when a deleted span still exists`
- `fails when deprecated msg.kind is still present in production source`
- `ignores excluded test files even when they contain invalid spans`

These fixtures act as in-test negative regression coverage and prevent the gate itself from silently degrading.
