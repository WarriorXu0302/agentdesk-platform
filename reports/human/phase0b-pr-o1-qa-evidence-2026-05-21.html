§3 备注 / 偏差说明 (disclosures)

项观察处置 / 解释
note Step 4 首次 curl /api/health 返回 curl: (52) Empty reply from server，重试后才得到 {"database":"ok"} 属预期行为：Grafana 容器 Started 后 plugin loading + provisioning 仍需要数秒；README §First-Run §Step 4 的 "等 ~10 秒后" 已点出。证据文件保留两次响应以如实记录。
note Step 10 后 docker volume ls -f name=muap_ 仅显示 2 个卷，没有 muap_phoenix_sim_data 正确：本次仅启动 prod stack，未启动 sim stack（obs:up:sim 未执行），因此 sim 卷从未创建。两 stack 互相独立，不影响 prod QA 闸门判定。
note Step 5 datasource type 字段值是 grafana-postgresql-datasource 而非 README 节录的 postgres 正确：Grafana 11.x 起官方 PostgreSQL plugin 的内部 type id 是 grafana-postgresql-datasource（与 datasource 配置 yaml 中的简写 postgres 是同一个 plugin 的两种引用），uid 与 readOnly 等核心字段都符合预期。
note 未在浏览器实际打开 Grafana UI 渲染占位 dashboard QA 表第 7 步要求 "≥ 1 条结果 · folder MUAP Observability"，已通过 search API 满足。浏览器渲染属于操作员日常使用范畴，不在 ADR-0009 §Acceptance 必填项内。
