§4 关键故事 · TDD 守门粒度修正 (scope guard from "all files" → "import-capable files")

⚠️ 守门测试一开始过分严格，连 ADR 自己都被它拦截

第一版 scripts/observability-bootstrap.test.ts 的 "keeps PR-O1 scope free from forbidden runtime imports" 用例 在 PR-O1 涉及到的每一个文件（含 .md、.yml、.json、.sql）里搜索 @opentelemetry/* 与 @arizeai/openinference-* 这些被本 PR 显式禁止 import 的包名。 结果：ADR-0009 自己（合法地）写出 "未来 PR-O2 将引入 @opentelemetry/*，本 PR 禁止" 这样的说明， 触发了守门测试 — 用文档解释为什么禁止 import 的字面量被错误等同于"真的 import 了"。

修复前（守门测试 BEFORE，过于粗暴）

// scripts/observability-bootstrap.test.ts (BEFORE)
it('keeps PR-O1 scope free from forbidden runtime imports', () => {
  const files = walkPrO1Files();
  for (const file of files) {
    const content = readFileSync(file, 'utf8');
    // ↓ 把 Markdown / YAML / JSON / SQL 全都纳入扫描
    expect(content).not.toMatch(/@opentelemetry\//);
    expect(content).not.toMatch(/@arizeai\/openinference-/);
  }
});

修复后（按"可 import 的文件扩展名"过滤）

// scripts/observability-bootstrap.test.ts (AFTER)
const IMPORT_CAPABLE_EXTS = new Set([
  '.ts', '.tsx', '.js', '.mjs', '.cjs', '.json',
]);

it('keeps PR-O1 scope free from forbidden runtime imports', () => {
  const files = walkPrO1Files()
    .filter(f => IMPORT_CAPABLE_EXTS.has(path.extname(f)));

  for (const file of files) {
    const content = readFileSync(file, 'utf8');
    expect(content).not.toMatch(/@opentelemetry\//);
    expect(content).not.toMatch(/@arizeai\/openinference-/);
  }
  // Markdown / YAML / SQL 可以合法地提到这些包名作为"defer 到 PR-O2/O3 的说明"，
  // 它们不构成 runtime import；守门只看真正能 import 的文件类型。
});

§7 ADR-0009 关键决策摘要 (9 条 · Accepted)

#决策选项判定理由
1 PR 拆分粒度 Option A · infra-only bootstrap 边界明确、可独立回滚；与 Oracle ruling 兼容；blast radius 最小

8 env 协同 host 端走 ADR-0008 .env.local.proposed 流程；compose 服务凭据独立 .env redaction policy 沿用；机器之间一致
