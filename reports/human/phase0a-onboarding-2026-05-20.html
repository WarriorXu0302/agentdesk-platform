§3 关键故事 · Header Merge 修复 (manual-QA-driven)

⚠️ 手工 QA 抓到的 wholesale-replace bug

自动测试全绿之后做 manual QA，跑 pnpm configure:enterprise-gateway --base-url http://localhost:8088，发现 lab desk 的 container.json 被改写后 canonical X-FrontLane-Source header 与 timeoutMs=30000 全部丢失。 根因：configure 把整个 enterpriseGateway 对象 replace 掉了，没有 merge。 这意味着 ADR-0008 设计里 "lab desk 用 X-FrontLane-Source 在 ERP gateway 侧识别身份" 的路径被静默拆毁。

修复前（旧实现）

// scripts/configure-enterprise-gateway.ts (BEFORE)
updateContainerConfig(folder, (config) => {
  config.enterpriseGateway = {
    baseUrl: args.baseUrl,
    timeoutMs: args.timeoutMs ?? undefined,
    defaultHeaders: Object.keys(args.headers).length > 0 ? args.headers : undefined,
    // ↑ 整个对象被 wholesale-replace
  };
});

修复后（merge semantics）

// scripts/configure-enterprise-gateway.ts (AFTER)
updateContainerConfig(folder, (config) => {
  const existingHeaders = config.enterpriseGateway?.defaultHeaders ?? {};
  const mergedHeaders = { ...existingHeaders, ...args.headers };
  const hasHeaders = Object.keys(mergedHeaders).length > 0;

  config.enterpriseGateway = {
    baseUrl: args.baseUrl,
    timeoutMs: args.timeoutMs ?? config.enterpriseGateway?.timeoutMs,
    defaultHeaders: hasHeaders ? mergedHeaders : undefined,
  };
  config.memoryMode = args.memoryMode;
  config.a2aSessionMode = 'root-session';
});

修复后 manual QA 输出 — groups/frontlane-lab-frontdesk/container.json

{
  "mcpServers": {},
  "enterpriseGateway": {
    "baseUrl": "http://localhost:8088",     // ← CLI 写入
    "timeoutMs": 30000,                      // ← canonical 保留
    "defaultHeaders": {
      "X-FrontLane-Source": "frontlane-lab-frontdesk"  // ← canonical 保留
    }
  },
  "packages": { "apt": [], "npm": [] },
  "additionalMounts": [],
  "skills": "all",
  "memoryMode": "erp",
  "a2aSessionMode": "root-session",
  "maxMessagesPerPrompt": 10,
  "resources": { "memoryMb": 1024, "cpus": 1, "pidsLimit": 512 }
}

回归测试

  • preserves pack-provided defaultHeaders and timeoutMs (ADR-0008 merge semantics) — seeded canonical → configure → 断言 X-FrontLane-Source + timeoutMs=30000 + CLI 新加的 x-tenant 三个都在
  • CLI --header overrides a same-key header from the existing config — seeded 旧 x-tenant: erp-old → CLI --header x-tenant=erp-new → 断言最终值为 erp-new
教训 · 类型检查 + 单元测试全绿不等于业务语义正确。Manual QA 这一步在宪法里强制是有原因的 — CLAUDE.md MANUAL_QA_MANDATE 是这次 catch 的根本来源。

§5 ADR-0008 关键决策摘要

决策选项判定理由
多 frontdesk 默认拓扑 双 desk 默认 + --frontdesks 显式 V1 / V2 灰度过渡期需要并存；运维一条命令拉起全部
lab desk 是否接 shared workers 不接 · 直连 ERP gateway worker 反向 destination 命名冲突；语义对齐 V1（lab desk = ERP 直驱）
canonical 文件入 git 方式 .gitignore 精确例外 禁止 git add -f；让默认 git 行为自然纳入 canonical pack
env 是否自动写 只写 .env.local.proposed + diff 报告 保留人工 redaction policy；<PLACEHOLDER> 不许被 agent 替换
configure 的 enterpriseGateway 行为 merge（addendum） manual QA 发现 wholesale-replace 会丢 pack-canonical header；CLI 同 key 覆盖、其余保留
--base-url 缺省 回退 ERP_GATEWAY_BASE_URL 让 setup:lab-frontdesk 在 env 就绪后能无人值守执行
